## Problem Statement

Website owners, SEO specialists, and developers need to understand and control how AI crawlers interact with their content. The landscape of AI-related web standards is fragmented: robots.txt (legacy), llms.txt (new, Sept 2024 spec by Jeremy Howard), ai.txt (emerging). No single free tool covers all of them in one place with generators, validators, and education.

## Product Vision

**llmstxt.codes** — the free, comprehensive AI Web Standards Hub. One URL to check everything, generate everything, and understand what each file does. Plus a WordPress plugin and Chrome extension to bring the tools where users already work.

## Core Features

### 1. Website — AI Readiness Scanner + Hub
- **URL Scanner**: Paste any domain, get an AI Readiness Score (0-100) with breakdown
  - robots.txt analysis (AI bot coverage)
  - llms.txt check (exists, valid, completeness)
  - ai.txt check (exists, valid)
  - sitemap.xml check (accessible, AI-optimized)
  - Meta robots tags scan (noai, noimageai directives)
  - HTTP headers check (X-Robots-Tag)
- **Shareable Reports**: `/report/{domain}` with visual dashboard
- **File Generators**: Guided wizards for llms.txt, robots.txt (AI rules), ai.txt
- **File Checker**: Upload or paste file content for validation (not just URL)
- **Learn Hub**: Educational pages for each file type (spec, playground, best practices)
- **Live Playground**: Edit and preview llms.txt/robots.txt in real-time

### 2. WordPress Plugin
- One-click llms.txt/ai.txt generator from WordPress admin
- Auto-generate from site content, pages, posts
- Settings page to configure AI crawler rules in robots.txt
- Scan button to check current AI readiness score
- Auto-update llms.txt when content changes
- Distribution: WordPress Plugin Directory (free)

### 3. Chrome Extension
- Right-click any page → "Check AI Readiness"
- Browser action popup showing quick score for current domain
- View/validate llms.txt, robots.txt, ai.txt for any site you visit
- Quick generator: create files based on the current site
- Side panel with detailed analysis
- Distribution: Chrome Web Store (free)

### 4. Research & Learn from Existing Sites
- Crawl and catalog sites that have implemented llms.txt
- Directory of real-world llms.txt examples
- Best practices derived from analyzing top implementations
- Competitive landscape of AI web standards tools

### Architecture Overview

```
┌──────────────────────────────────────────────────────────────┐
│                    Browser (Client)                          │
│                                                              │
│  index.astro → scanner.ts → parsers.ts → scoring.ts         │
│       │              │                       │               │
│       │         fetch proxy             compute score        │
│       │              │                       │               │
│       ▼              ▼                       ▼               │
│   User Input    /api/v1/fetch          ScoreResult           │
│                     │                       │               │
│                     │                       │               │
│                     ▼                       ▼               │
│             /api/v1/save              Display Results        │
└───────┬─────────────┬─────────────────────────────────────────┘
        │             │
        ▼             ▼
┌──────────────────────────┐    ┌─────────────┐
│ Cloudflare Workers/Pages │    │ Cloudflare   │
│                          │    │ D1 (SQLite)  │
│ - Fetch proxy (4 files)  │    │ - scans      │
│ - SSRF validation        │    │ - alerts     │
│ - 3s timeout, 500KB cap  │    │ - metrics    │
│ - Save to D1 + R2        │    │ - directory  │
└──────────────────────────┘    └─────────────┘
                                       │
                                ┌──────┴──────┐
                                │ Cloudflare   │
                                │ R2 (Object)  │
                                │ Full JSON    │
                                │ results      │
                                └─────────────┘
```

### Data Flow Analysis

1. **Scan Flow**: User input → `normalizeDomain()` → `validateDomain()` (SSRF check) → `buildFetchUrls()` → Worker fetches 4 files in parallel → client parses → `computeScore()` → save to D1/R2 → display
2. **Report Flow**: `/report/{domain}` → query D1 for latest scan → SSR render with score + breakdown
3. **Score API**: `/api/v1/score/{domain}` → query D1 → return JSON (read-only cache)

**Correctness**: The scan flow is sound. Client-side parsing eliminates Workers CPU pressure. The 4-file parallel fetch uses `Promise.allSettled` correctly — one failure doesn't block others.

### Scoring Algorithm Issues

**Current scoring inflates scores because meta and headers are always 0:**
```
overall = llms*0.30 + robots*0.25 + sitemap*0.15 + ai*0.10 + 0*0.10 + 0*0.10
```
Effective max score is 80 (not 100). A perfect site gets 80, not 100.

```
llmstxt-codes-wp/

├── templates/
│   └── llms-txt.php           # Template for serving /llms.txt
└── readme.txt                 # WP Plugin Directory readme
```

**Key technical decisions:**
- Serve llms.txt via WordPress rewrite rule (flush on activation)
- Auto-generate from post types: pages (always), posts (configurable), CPTs (configurable)
- Store generated content in `wp_options` (no custom table)
- Cache score result for 24h in transient
- Minimum PHP 8.0, WP 6.0
- No external dependencies beyond `wp_remote_get()`

### Chrome Extension Architecture

```
llmstxt-codes-extension/
├── manifest.json             # Manifest v3
├── background.js             # Service worker: handle right-click, badge
├── popup.html + popup.js     # Browser action popup (quick score)
├── content.js                # Inject score badge on viewed pages (optional v2)
├── icons/                    # Extension icons (16, 32, 48, 128)
└── styles/popup.css          # Popup styles (match website design)
```

**Key technical decisions:**
- Use `activeTab` permission (no broad host permissions)
- `contextMenus` permission for right-click "Check AI Readiness"
- Fetch score from `https://llmstxt.codes/api/v1/score/{domain}`
- Cache scores in `chrome.storage.local` with 24h TTL
- Popup is 320×480px, shows score gauge + breakdown
- No content script injection in v1 (avoids permission concerns)
- Manifest v3 only (no v2 APIs)
